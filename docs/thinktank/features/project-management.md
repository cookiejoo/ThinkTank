# 项目管理

本文档详细说明 ThinkTank 的项目和分组管理功能。

## 项目概念

### 项目类型

ThinkTank 支持两种项目模式：

#### 1. 在线编辑模式 (Edit Mode)

- **特点**：
  - 完整的在线编辑能力
  - 支持创建、编辑、删除文件
  - 拖拽排序
  - 个性化数据支持
- **适用场景**：
  - 内部知识库
  - 快速迭代的文档
  - 需要协作的内容

#### 2. Git 镜像模式 (Git Mirror Mode)

- **特点**：
  - 只读访问
  - 自动 Git 同步
  - 支持版本浏览
  - 支持指定分支和子目录
- **适用场景**：
  - 产品官方文档
  - API 参考文档
  - 需要与代码同步的内容

## 项目操作

### 创建项目

**入口**：管理后台 → "Add Project"

**流程**：

1. 填写项目信息
2. 选择模式（Edit 或 Git）
3. 如果是 Git 模式，填写 Git 配置
4. 选择或创建分组
5. 点击 "Create Project"

**API**：`POST /api/projects`

### 编辑项目

**入口**：管理后台 → 点击项目卡片

**可编辑内容**：

- 项目描述
- 项目分组
- Git 配置（模式、仓库、分支、同步间隔）

**API**：`PUT /api/projects/[id]`

### 删除项目

**入口**：管理后台 → 悬停项目卡片 → 删除图标

**确认对话框**：

- 是否永久删除（Permanently delete）

**删除模式**：

- **逻辑删除**（默认）：项目在 UI 中隐藏但文件保留
- **物理删除**：从磁盘删除整个项目目录

**API**：`DELETE /api/projects/[id]?physical=false`

### 测试连接

**入口**：项目设置 → "Test Connection" 按钮

**功能**：

- 使用当前配置测试 Git 连接
- 立即反馈成功或失败
- 支持使用 `********` 占位符（已保存的 Token）

## 分组管理

### 分组概念

分组用于组织和分类项目：

```
核心项目
├── ThinkTank Docs
├── API Reference
└── User Guide

实验性项目
├── Project A
└── Project B
```

### 分组操作

#### 创建分组

- **方式 1**：创建项目时输入新分组名
- **方式 2**：管理后台 → "Group Management" → "Add Group"

#### 分配项目

- **创建项目时**：通过下拉框选择分组
- **编辑项目时**：在设置页面修改分组

#### 删除分组

**入口**：管理后台 → "Group Management" → 删除图标

**限制**：只有空分组可以删除（防止误操作）

**恢复**：删除分组后，项目自动移到"其他项目"

### 分组配置

位置：`docs/.thinktank/groups.json`

```json
[
  {
    "name": "核心项目",
    "projects": ["thinktank-docs", "api-reference"],
    "isDeleted": false
  }
]
```

## UI 展示

### 管理后台

**入口**：`/admin`

**功能**：

- 项目卡片列表（按分组分组显示）
- 分组管理标签页
- 用户管理标签页

### 项目预览页

**入口**：`/preview/[id]`

**特点**：

- 纯粹的阅读体验
- 无编辑功能
- 公开访问（无需登录）

### 项目详情页

**入口**：`/project/[id]`

**布局**：

- 左侧：文件树导航
- 右侧：编辑器或仪表盘

## 项目元数据

### 项目描述

- 显示在首页仪表盘
- 可行内编辑（点击编辑图标）
- 支持实时保存

### 项目模式标识

- Git 模式：文件树禁用编辑功能
- Edit 模式：显示所有编辑操作

### 分组徽章

- 显示在项目卡片
- 显示分组名称
- 不同分组可用不同颜色（未来功能）
