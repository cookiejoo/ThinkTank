# 项目概述

## 什么是 ThinkTank？

ThinkTank 是一个为技术团队设计的、文件原生的混合模式知识中心。它无缝融合了"在线协作编辑"与"Git 仓库镜像"两种工作流，旨在提供极致的文档管理与阅读体验。

## 核心理念

### 三层架构设计

ThinkTank 采用清晰的分层架构设计：

```
┌─────────────────────────────────────────────────────────┐
│              体验层 (Experience Layer)              │
│  ┌──────────────────────┐ ┌──────────────┐   │
│  │   在线编辑模式     │ │  Git镜像模式  │   │
│  │   (协作/实时编辑)    │ │  (只读/同步)   │   │
│  └──────────────────────┘ └──────────────┘   │
└─────────────────────────────────────────────────────────┘
                       │ 控制
                       ↓
┌─────────────────────────────────────────────────────────┐
│              逻辑层 (Logic Layer)                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ 项目管理  │ │ 分组管理  │ │ 权限控制  │ │
│  └──────────┘ └──────────┘ └──────────┘ │
│  ┌──────────┐ ┌──────────┐              │
│  │ Git同步  │ │ 个性化数据│              │
│  └──────────┘ └──────────┘              │
└─────────────────────────────────────────────────────────┘
                       │ 持久化
                       ↓
┌─────────────────────────────────────────────────────────┐
│              物理层 (Physical Layer)              │
│  ┌──────────────────────────────────────────┐       │
│  │          文件系统 (Source of Truth)        │       │
│  │  Markdown 文档 + JSON 配置文件          │       │
│  └──────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────┘
```

### 数据源真理 (Source of Truth)

ThinkTank 的核心设计哲学是以磁盘文件系统为唯一数据源：

- **物理文件**：所有 Markdown 文档和目录结构是数据的核心
- **配置驱动**：通过轻量化的 JSON 配置文件管理元数据
- **无数据库依赖**：不需要任何关系型数据库

### 两种工作模式

#### 1. 在线编辑模式 (Edit Mode)

适用于需要快速迭代、团队协作的内部知识库。

- **特点**：
  - 完整的在线编辑能力
  - 支持 Markdown 实时编辑和预览
  - 文件拖拽排序和目录管理
  - 支持创建、编辑、删除文档
- **使用场景**：
  - 团队内部知识库
  - 快速迭代的文档
  - 需要在线协作的内容

#### 2. Git 镜像模式 (Git Mirror Mode)

适用于需要严格版本控制、与代码一同发布的产品文档。

- **特点**：
  - 只读访问，不可在线编辑
  - 自动定时同步远程 Git 仓库
  - 支持浏览 Git 标签（版本）
  - 支持指定分支和子目录
- **使用场景**：
  - 产品官方文档
  - API 参考文档
  - 需要与代码版本同步的内容

## 核心功能

### 权限体系 (RBAC)

- **游客 (Guest)**：只读访问，不可编辑
- **普通用户 (User)**：基于项目授权的读写访问
- **管理员 (Admin)**：完全控制权

详细说明见 [权限体系文档](../architecture/auth-rbac.md)。

### 项目分组管理

- 将项目组织到不同的分组中
- 支持分组的增删改查
- 未分组项目显示在"其他项目"

### 个性化功能

- **收藏夹**：收藏重要文档
- **最近访问**：快速返回最近查看的文档
- **数据隔离**：每个用户的个性化数据独立存储

### 版本管理 (Git Tags)

Git 模式下支持版本浏览：

- 自动获取所有 Git 标签
- 支持按版本切换内容
- 版本切换时禁用编辑功能

## 系统入口

```
/                    ┌──────────────┐
│              /    │  公开项目主页
│   (游客/用户可见)  │  浏览所有项目
└───────────────────┘

/admin              ┌──────────────┐
│   管理后台    │  仅登录用户可见
│   (需登录)      │  管理项目和分组
└───────────────────┘

/project/[id]        ┌──────────────┐
│   项目详情页    │  侧边栏 + 内容区
│   (需权限)       │  浏览和编辑文档
└───────────────────┘

/preview/[id]        ┌──────────────┐
│   项目预览页    │  纯粹的阅读体验
│   (公开访问)      │  不可编辑
└───────────────────┘
```

## 项目优势

相比其他知识库和文档系统：

1. **独一无二选一**：同时支持在线编辑和 Git 镜像
2. **轻量级架构**：无数据库依赖，部署简单
3. **开发者友好**：Git 原生集成，符合开发者工作流
4. **数据透明**：所有数据都是可读的文件，便于备份迁移
